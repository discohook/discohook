## New
- The backups management page now has a searchbar
- Redesigned the settings menu for clearer navigation
- Added warning for components duplicated on multiple backups, helping to reduce or eliminate issues like [#65](https://github.com/discohook/discohook/issues/65)
- Added discord.py code generation for components-based messages
- Redesigned the flow editor for more visual clarity and less clutter
  - Actions are no longer translucent or blue
  - The two branches of a "Check" action are delegated into tabs, only one visible at a time, rather than being stacked vertically
  - Indentations and thread lines help to illustrate the order of events
  - Actions are now collapsible
- If you open the editor with invalid data, Discohook will let you load it anyway
- There is now a dropdown next to the send/edit button which will prompt you to add or remove message links (enabling or disabling "edit mode")

## API
- Advancements have been made concerning a previous announcement about more `data.targets` types (not Discord Webhook). You may now specify a query parameter to opt-in to receiving these new types when resolving share links, though it is currently an experiment and you may not see any such data for some time.
  - [Get Share Link](https://docs.discohook.app/api-7479151)
  - [QueryDataTarget](https://docs.discohook.app/schema-6693463)

## Fixed
- Fixed bug where components in components-based messages would not be properly registered after being sent ([#56](https://github.com/discohook/discohook/issues/56))
- Fixed bug where some backups would not be properly recognized in the backups menu when loaded ([#66](https://github.com/discohook/discohook/issues/66))
- If something goes wrong while submitting your messages, Discohook will automatically reset the send button after 2 minutes so you don't have to refresh the page
- Corrected various component & editor styling

## Updates
- 🇹🇼 New Traditional Chinese translation is available
- 🇧🇷 85 new Brazilian Portuguese strings

**Security Notice:** On June 16, an exploit was patched regarding mistakenly accessible roles by users who are able to author flows. No action is required. [Read more](discohook://news/security/2026-06-16).

**Privacy Notice:** Discohook's [Privacy Policy and Terms of Service](discohook://legal) have been updated for improved clarity via changes to structure & wording.

## New

**Attachments & files overhaul!**
- Discohook now displays your draft attachments in a row, just like in the Discord app, and presents quick options for managing them.
  - -# **TIP:** To copy an attachment's name, click on the filename.
- Attachments can be rearranged without removing them
- You can reuse existing attachments within embeds/galleries
- If you refresh the page, it will alert you about missing files

**User experience**
- If a Discord error is encountered, the error page will be much more descriptive and may detail which permissions are missing, if applicable
- When loading the default Discohook page, you'll be prompted to try editing standard or components-based messages (for new users), start a blank message, or load a recent backup (if logged in)
- There is now a total character counter for components-based messages
- Game mentions are now supported in the preview with the format `<@$id>`. Discord only supports this in standard content, not embeds nor display components
- "Large Image" and "Thumbnail" labels once again include the term "URL" to reduce confusion (you can still add local files)

## Fixed
- Fixed channel select not appearing for webhooks accessed from the server page ([#87](https://github.com/discohook/discohook/issues/87))
- Fixed webhook list on the server page changing dramatically after saving a webhook
- Fixed incorrect syntax in discord.py code generator ([#85](https://github.com/discohook/discohook/issues/85))
- Fixed incorrect end parenthesis in discord.py code generator ([#94](https://github.com/discohook/discohook/issues/94))
- Fixed absence of avatar fallback on server homepage
- Fixed incorrect invite link permissions

## Updates
- Components-based messages are now considered stable (no more info box)
- Changelogs are now separate from guides
